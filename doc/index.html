The CH2O formalization

By Robbert Krebbers, Radboud University Nijmegen

This is an ongoing development. If you are looking for the Coq sources corresponding to my papers or forks of this development, please take a look here.

About

The CH2O project aims at formalizing novel features of the ISO/ANSI C11 standard of the C programming language. This Coq development includes an operational and axiomatic semantics for a fragment of a C-like language supporting:

  • Non local control flow (return and goto).
  • Block scoped variables and pointers to these.
  • Mutual recursive function calls.
  • Dynamically allocated memory (alloc and free).
  • A powerful permission system based on fractional permissions.
  • Non-deterministic expressions with multiple side-effects.
  • A strict interpretation of sequence points.
  • An axiomatization of machine integers that takes overflow seriously.
  • Big and little endian two's complement integer implementations parametric in the bit length of bytes.

The operational semantics is given by a small step reduction semantics using a zipper like data structure to accurately model allocation and deallocation of block scoped variables. For example, our semantics will correctly assign undefined behavior to uses of non-local control flow that jump out of scope:

  int f(void) {
    int *p = 0;
    l : if (p) return (*p);
    { int j = 10; p = &j; goto l; }
  }

The axiomatic semantics is defined semantically with respect to the operational semantics. We prove that it satisfies the standard rules of separation logic and novel rules for our specific features. The rules to reason about non-deterministic expressions are inspired by the separation logic rules for parallelism, and use a permission system based on fractional permissions.

Goals

To get more closely to the actual C11 standard we intend to implement the following features in future versions:

  • The C type system including structs, arrays and unions.
  • The non-aliasing restrictions introduced by C99 and C11 (effective types in particular).
  • A simple verification condition generator inside Coq allowing us to verify simple programs.
  • An embedding of the CompCert semantics into our semantics.

A work-in-progress version of the first two goals can be found here.

Table of contents

Prelude

  • base: interfaces, notations, and general theorems.
  • tactics: general purpose tactics.
  • proof_irrel: facts about proof irrelevant types/propositions.
  • decidable: theorems on decidable structures.
  • orders: theorems on ordered structures.
  • option: miscellaneous definitions and theorems on the option data type.
  • list: miscellaneous definitions and theorems on lists.
  • vector: miscellaneous definitions and theorems on vectors.
  • numbers: miscellaneous facts on nat, N and Z.
  • ars: theorems and definitions on abstract rewriting systems.
  • collections: definitions, theorems, and tactics for collections.
  • fin_collections: theorems on finite collections.
  • listset: finite set implementation using unordered lists.
  • fresh_numbers: generation of fresh numbers.
  • prelude: exports all of the above.

Miscellaneous

  • contexts: general purpose definitions on contexts/zippers.
  • listset_nodup: finite set implementation using unordered lists without duplicates.

Finite maps

  • fin_maps: an interface and theorems for finite maps.
  • fin_map_dom: an interface and theorems for the domain function of finite maps.
  • natmap: finite maps indexed by nat.
  • pmap: finite maps indexed by positive.
  • nmap: finite maps indexed by N.
  • mapset: finite maps give rise to finite sets with extensional equality.

Values

  • abstract_integers: an axiomatization of machine integers.
  • integers: an implementation of big and little two's complement machine integers.
  • abstract_values: values parametrized by arbitrary integer implementations.
  • values: values instantiated with concrete machine integers.

Memory

  • abstract_permissions: an axiomatization of permissions.
  • abstract_memory: the memory parametrized by arbitrary permissions and integer implementations.
  • empty_permissions: an implementation of permissions using the unit type.
  • frac: fractional permissions.
  • seqfrac: lockable fractional permissions.
  • permissions: readonly, writable, freeable, and lockable fractional permissioms.
  • memory: an instantiation of the memory model using concrete permissions.

Operational semantics

  • expressions: expressions and their decomposition into contexts.
  • statements: statements and program contexts.
  • state: the execution state.
  • smallstep: the small step operational semantics.
  • expression_eval: evaluation of side-effect free expressions.
  • expression_eval_correct: correctness of evaluation of side-effect free expressions.

Axiomatic semantics

  • assertions: assertions of the axiomatic semantics.
  • axiomatic: axiomatic semantics using separation logic.
